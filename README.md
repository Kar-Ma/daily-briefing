# Daily Briefing

A personal morning briefing web app. Open it with your coffee and see weather, headlines, and your calendar for the day on a single page.

## What it shows

- **Weather** for your location, plus a one-line clothing recommendation generated by an LLM
- **Top 3 headlines** curated from Brave Search and rewritten by Claude (or GPT) in a punchy, contextual style
- **Today's calendar** merged across multiple Google Calendars with color-coded sources, plus a friendly one-line closing note
- **Auto-refreshes once per calendar day** and caches; ongoing cost is roughly **$0.50–1.00/month** in LLM calls

## Tech stack

Next.js 16 (App Router) · TypeScript · Tailwind CSS · Vercel · Anthropic & OpenAI APIs · Brave Search · Open-Meteo

## Deploy your own

This project is designed to be self-hosted. Each user runs their own copy with their own API keys and calendars. There's no shared infrastructure or signup.

### Prerequisites

- Node.js 18+ and npm
- A GitHub account (for forking & deploy)
- A Google Calendar account (for the calendar section)
- API keys (free tiers are sufficient for personal use):
  - [Brave Search API](https://api.search.brave.com) — for news
  - [Anthropic API](https://console.anthropic.com) — for LLM curation (~$5 minimum credit)
  - [OpenAI API](https://platform.openai.com) — optional, only needed if you want GPT models

### Steps

1. **Fork this repo** to your GitHub account.

2. **Clone and install** dependencies:
   ```bash
   git clone https://github.com/YOUR_USERNAME/daily-briefing.git
   cd daily-briefing
   npm install
   ```

3. **Set up environment variables.** Copy the example file and fill it in:
   ```bash
   cp .env.example .env.local
   ```
   Open `.env.local` in your editor. Each variable is documented inline. Key ones:
   - `LOCATION_LAT`, `LOCATION_LON`, `LOCATION_LABEL` — your default location
   - `CALENDAR_1_URL` (and `_NAME`, `_COLOR`) — your Google Calendar's "Secret address in iCal format" (find it in Google Calendar → calendar settings → Integrate calendar). Add `CALENDAR_2_*`, `CALENDAR_3_*`, etc. for additional calendars.
   - `BRAVE_API_KEY`, `ANTHROPIC_API_KEY` — your API keys
   - `DEFAULT_LLM_MODEL` — defaults to `claude-haiku-4-5-20251001` (cheap). Change to `claude-sonnet-4-6` for higher quality summaries.

4. **Run locally**:
   ```bash
   npm run dev
   ```
   Open [http://localhost:3000](http://localhost:3000).

5. **Deploy to Vercel** when you're happy with it:
   - Push your fork to GitHub
   - Import the repo at [vercel.com/new](https://vercel.com/new)
   - In the import flow, click "Import .env" and select your `.env.local` file (note: on macOS Finder, press `Cmd+Shift+.` to show hidden files)
   - Click Deploy. You get a public URL on the free tier.

6. **Add to your phone's home screen** for an "app" feel: open the deployed URL in Safari/Chrome on your phone → Share → Add to Home Screen. The dawn icon will appear on your home screen.

## Configuration

### Per-task LLM model selection

You can use a different model for each LLM-driven task. For example: cheap Haiku for the calendar note, Sonnet for the news, GPT for the clothing rec.

```env
DEFAULT_LLM_MODEL=claude-haiku-4-5-20251001    # used when a task-specific model isn't set
NEWS_MODEL=claude-sonnet-4-6                    # override for news only
CLOTHING_MODEL=                                 # blank → uses default
CALENDAR_MODEL=                                 # blank → uses default
```

Provider is auto-detected from the model name (`claude-*` → Anthropic, `gpt-*` or `o*` → OpenAI). To use OpenAI, set the model to e.g. `gpt-5.4-mini` and make sure `OPENAI_API_KEY` is filled in.

### Multiple calendars

Add as many calendars as you want by incrementing the number. The loop stops at the first missing `_URL`.

```env
CALENDAR_1_URL=https://...
CALENDAR_1_NAME=Work
CALENDAR_1_COLOR=blue

CALENDAR_2_URL=https://...
CALENDAR_2_NAME=Personal
CALENDAR_2_COLOR=green
```

Available colors: `blue`, `green`, `purple`, `rose`, `amber`, `orange`, `cyan`, `fuchsia`, `teal`, `indigo`.

## Architecture

```
src/lib/
  ├── llm.ts          — provider routing (Anthropic / OpenAI)
  ├── prompts.ts      — all LLM prompts in one place
  ├── location.ts     — location resolution (cookie + env fallback)
  ├── weather.ts      — Open-Meteo fetch + clothing recommendation
  ├── news.ts         — Brave Search fetch + LLM curation
  └── calendar.ts     — iCal parsing + closing note

src/app/
  ├── page.tsx               — composition & JSX (no business logic)
  ├── actions.ts             — server actions (e.g. set location cookie)
  ├── location-switcher.tsx  — client component
  └── apple-icon.tsx         — generated home screen icon
```

Each LLM-driven feature follows the same pattern: a prompt function in `prompts.ts`, a task function in the relevant `lib/` file, both wrapped with `unstable_cache` so they only run once per calendar day. Adding a new card means adding one new lib file and following the same recipe.

## Cost

For typical personal use (one or two visits per day):

| Item | Cost |
|---|---|
| Open-Meteo (weather) | Free |
| Google Calendar (iCal URL) | Free |
| Brave Search | Free tier covers it |
| Anthropic — Sonnet 4.6 for news, Haiku 4.5 elsewhere | ~$0.50–1.00/month |
| Vercel hosting | Free tier |

The cache is the cost saver: each LLM-curated section runs once per calendar day, then serves from cache for the rest of the day.
